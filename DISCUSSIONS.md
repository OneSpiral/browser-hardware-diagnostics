# Discussions Guide

Use GitHub Discussions for open-ended collaboration around the public toolkit.

## Good discussion topics

Discussions are a good place for:

- design ideas
- browser compatibility questions
- diagnostics API design tradeoffs
- measurement methodology questions
- contributor onboarding questions
- roadmap exploration before opening a scoped issue

## What should become an issue

Open an issue instead when you already have a clear, actionable item such as:

- a reproducible bug
- a concrete feature request
- a documentation task with clear scope

## What should not go in Discussions

Do not post sensitive security issues in Discussions.

If something may be security-sensitive, follow the private reporting path in [SECURITY.md](./SECURITY.md) instead of opening a public issue or discussion.

## Goal

The purpose of Discussions is to help the community refine technical ideas before they turn into issues, pull requests, and reusable browser diagnostics capabilities.
