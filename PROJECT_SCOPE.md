# Project Scope

## Classification

- Type: solution
- Visibility: public
- Workspace bucket: solutions
- Private eval assets: forbidden

## Workspace convention

Top-level workspace folders should stay lowercase:

- `solutions/`
- `apps/`
- `labs/`

Visibility is a repo property, not a folder name.

## Role in the portfolio

This repository is a reusable technical solution.

- branded deployments belong in apps
- exploratory prototypes can begin in labs
- mature reusable work can graduate from labs into solutions

## Boundary reminder

Do not store private evaluation assets here.
That includes hidden fixtures, golden datasets, browser / device regression suites, and tolerance thresholds.
